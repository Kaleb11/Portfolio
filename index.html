Ride Benefit System(Booking Management Service)

RIDE is the biggest taxi booking platform in Ethiopia & Djibouti. Operate in Addis Ababa.

This company has a software development tech company at Addis Ababa to develop RIDE hailing supportive applications. I was hired as a backend developer and I developed RIDE Benefit System. Let me describe you about this system in detail

RIDE Benefits platform developed for browser and mobile application to beneficiaries. The system serves for registered company platform, ride company platform and mobile application platform for beneficiaries. The system helps organizations plan and administer benefits packages of the RIDE company and ensure the beneficiaries access to those benefits from the parent companies. RIDE Benefits systems are used to manage benefits provided by the partner companies, such as insurance plans and stock options, and facilitate open enrollment periods through an administrative dashboard. Organizations also use these programs to view and adjust benefits packages. The RIDE Benefits packages frequently offer beneficiary portals and self-service options to enable beneficiaries to view, search benefit packages provided by the partner companies. The RIDE Benefit System shall be provided and managed by RIDE Company, different organizations( companies ) can be registered on to the RIDE Benefit System as partners to get many customers for their services and products from the RIDE company.

The system has:-

  • OAuth Integration: Integrated with their company OAuth server
  • User authentication and authorization: Implemented secure authentication and authorization for ride drivers to access the booking system.
  • Implemented Booking system: a system to allow ride drivers to book appointments for vaccinations and insurance coverage.
  • Appointment scheduling: Developed a system to with schedule appointments for vaccinations and insurance coverage based on availability and driver preferences.
  • Notification system: Created a system to send notifications to ride drivers regarding their vaccinations and insurance coverage, including appointment reminders and updates on their status.
  • Payment integration: Integrated a payment gateway to allow ride drivers to pay for their vaccinations and insurance coverage.
  • Reporting and analytics: Developed a system to generate reports and analytics on ride driver vaccination and insurance coverage for the project managers and administrators.
  • Tracking: Registered company track their booking data with various mechanism and the RIDE company itself has a tracking system for overall system.

ERP System for Franckon Company

The Franckon PLC ERP System is a comprehensive system that I have developed for managing human resources, authentication and authorization, and role-based permissions. The system has over 6 different roles, including Super Admin, Purchaser, Site Engineer, Main Store Keeper, and Store Keeper 2. The system has a robust role, permission, and notification system, allowing for easy management of employees with different roles.

Features of the Franckon PLC System:

  • Project Management: Super Admin is responsible for adding projects and assigning Site Engineer employees for each project. They have the authority to approve ordered items and control the entire system.
  • Ordering System: Site Engineers can order items for their assigned projects.
  • Item Approval: Main Store Keeper checks whether approved items are available in the store or need to be purchased. They have the authority to assign the items to be purchased or obtained from the store.
  • Purchasing System: Purchaser has the responsibility to purchase items, adding purchased company name, and resetting images of the items. They receive orders for items to be purchased and buy them accordingly.
  • Delivery Management: Store Keeper 2 is responsible for delivering items to Site Engineers. They have access to both bought and stored items and deliver them to the appropriate engineers. Upon delivery, they can tick the delivered checkbox, and the status of the item changes to 'Completed.' If the item is undelivered, they can tick the undelivered checkbox, and the status of the item changes to 'Failed.'
  • Notification management
The Franckon PLC ERP System is a highly effective tool for managing human resources, item ordering, delivery, and purchasing. It is user-friendly, with a clean interface, and a robust system for managing different roles, permissions, and notifications.

Blog System for Admins and Visitors

I am happy to share you my blog development experience. I developed this blog system in both ways to the admin dashboard and the user interface. This blog website typically consists of two main components: the frontend and the backend. The frontend is the part of the website that users interact with, while the backend is the behind-the-scenes system that powers the website's functionality. In a blog website, the frontend typically includes the user interface for reading and interacting with blog posts, while the backend includes the content management system (CMS) for creating, editing, and managing blog posts.

When I am developing a blog website I had considered the different roles that users may have in the system. The main roles in a blog website are typically the main admin, editor, moderator, and user.

  • Main Admin: The main admin is the highest level of authority in the blog website. They have access to all features and functionality, including the ability to create and manage other user accounts. The main admin is responsible for setting up the website and ensuring that it is running smoothly.
  • Editor: The editor is responsible for creating and editing blog posts. They have the ability to publish, edit, and delete blog posts, as well as manage categories and tags.
  • Moderator: The moderator is responsible for managing comments on blog posts. They have the ability to approve or delete comments and ban users who violate the website's commenting policy.
  • User: The user is the most basic level of user in the system. They can read blog posts, leave comments, and interact with the website in various ways.
When I am developing this blog website, I designed the system with these different roles in mind. This typically involves implementing user authentication and authorization to ensure that each user has access to the appropriate features and functionality based on their role in the system. In terms of the backend development process, there are several key steps I involved in building a blog website:
  • Setting up the database: The first step is to design and set up the database schema. This typically involves defining tables for blog posts, categories, tags, comments, and user accounts.
  • Creating the CMS: Once the database is set up, the next step is to build the CMS. This involves creating user interfaces for creating, editing, and managing blog posts, categories, and tags.
  • Implementing user authentication and authorization: The next step is to implement user authentication and authorization to ensure that users can only access the features and functionality that they are authorized to use.
  • Building the frontend: Once the backend is complete, the final step is to build the frontend of the website. This typically involves creating user interfaces for reading and interacting with blog posts, as well as implementing features like search, pagination, and commenting. Overall, developing a blog website had gave me a great project experience as a backend developer.

Multiple Image Resizer with Different Format

Overview:
My image resizer project is a backend application that allows users to resize images based on their desired dimensions. The user can input the size of the image in pixels and the proportion they want to maintain while resizing. The application then resizes the image and returns it to the user.

Functionality:
The main functionality of the image resizer project is to take an image as input and resize it according to the user's specifications. The user can specify the size of the image in pixels, as well as the proportion they want to maintain while resizing. This ensures that the image remains proportional, regardless of its new size. The image resizer project can handle a variety of image file types, including PNG, JPEG, and GIF. Once the user submits the image to be resized, the application uses an image processing library to resize the image according to the user's specifications. The resized image is then returned to the user, either as a downloadable file or as a direct link to the image. The application also allows users to save their resized images to their account for future use.

Architecture:
The application receives user requests through HTTP requests, which are handled by a server running the backend framework. The image processing library used to resize the images is integrated into the application, either as a third-party library or as a custom module developed specifically for the project. The library takes the user's image file as input and resizes it according to the specified dimensions and proportions. The resized image is then returned to the user through an HTTP response, either as a downloadable file or as a direct link to the image. The application can also store the resized images in a database, allowing users to access them later. User Interface: The user interface of the image resizer project is designed to be simple and intuitive. Users are prompted to upload an image and specify the desired size and proportion. The user can also choose to save the resized image to their account for future use. Once the user submits their request, the application processes the image and returns it to the user. The user can then download the resized image or copy the link to use it elsewhere.
